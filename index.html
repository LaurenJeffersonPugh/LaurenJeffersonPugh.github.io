Summer Project 2021 (Unity)

During the summer in 2021, I was involved in a game jam project with friends to develop a game which runs on browser, PC and mobile using Unity. The aim of the game is to get the RAT to the goal of each level. We spent 1 week developing this game. The team to develop this game had multiple sub-teams focused on different parts of the development. This included: game design team, art and music team, and the programming team.

I was a member of the programming team. I was involved in the rotation of the camera/gravity, the level manager, level progression, mobile controls and bug testing.

Paint Shooter Prototype - Master's Team Project (Unity)

For our Master's Team Project module, we were tasked with creating a paint shooter game (like Splatoon) for the PC and PS4, using our own engine. This video shows the prototype that was created using Unity. For the prototype, I was involved in the creation of the GUI, the player health, player damage, refilling ammo, destructable objects, and bug testing.

The aim of the game is to destroy the other team's paint reservoir. The paint is both the players' health and ammo. Each team have their own paint colour, if a player steps on the enemy team paint, they take damage. Players can refill their paint at their base paint reservoir. Players can paint any object and also have the ability to wall run/climb walls.

Procedural Generation of Terrain with focus on Creating a Realistic World (Unity)

For my stage 3 dissertation, I researched popular techniques for procedurally generating terrain, well-known biome-type models, and suitable game engines for generating terrain in order to create an application that allows for the realistic generation of terrain. I developed an application and performed testing to determine how effective it was at generating realistic terrain using Unity. The developed application generates terrain with biomes based on the terrain temperature and moisture levels.

Advanced Game Technologies (C++)

For my Advanced Game Technologies module during my Master's year, I was tasked with creating a game with 2 levels. The first level was to show physics implementation and the second level was to show state machines and pathfinding.

The first level shows implementations of collision detection, collision resolution, movement via forces, impulses, constraints, friction and gravity.

The 2nd level shows implementations of A* pathfinding and state machines.

The game uses pushdown automata for each level/menu screen.

Advanced Graphics for Games (C++/OpenGL)

For my stage 4 Advanced Graphics for Games module, I created an island scene using C++ and OpenGL. For this project I made the renderer and each shader from scratch. The shape of the island is generated from a heightmap that is passed into the C++ program. It features an automatic camera, a moving sun, animated meshes, moving water, rain, and a full screen blur post processing effect.
